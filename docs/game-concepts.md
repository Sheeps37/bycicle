# 老八骑车小游戏构思

## 核心气质

这张图最强的笑点不是“骑得快”，而是“明明快失控了还在努力骑”。所以玩法应该围绕平衡、摇晃、临门一脚的险象环生，而不是传统竞速。

## 方向一：老八平衡车挑战

玩家用左右键控制重心，用空格或上键蹬车。角色会持续左右晃，速度越快越难稳住。保持在平衡区内可以连击加分；倾斜过大就会进入“表情失控时间”，再不救回来就摔车。

适合做第一版，因为只需要一个人物抠图素材、一个滚动地面、障碍物和一个平衡条，就能有明确挑战。

## 方向二：车筐保卫战

车筐里放奶茶、书包、鸡蛋之类的道具。玩家骑车躲砖缝、减速带、路人影子，车身越晃，筐里的东西越容易飞出去。目标不是最快到达，而是“到终点时还剩多少东西”。

这个方向更偏物理喜剧，后续可以加入弹跳道具和慢动作回放。

## 方向三：校园学车模拟

关卡从空旷人行道开始，逐渐加入窄道、转弯、红线区域、突然出现的路障。每关有教练评价，比如“今天至少没撞到树”“车会走，人也还在”。

适合做成多关卡小品，但第一版内容量会稍大。

## 推荐 MVP：老八平衡车挑战

第一版目标：做一个 60 秒内尽量骑远、尽量不摔的网页小游戏。

核心循环：

1. 玩家按上键/空格蹬车加速。
2. 角色根据速度、地面起伏和随机扰动不断倾斜。
3. 玩家按左右键把倾斜角拉回安全区。
4. 路上出现砖缝、减速带、饮料瓶等障碍。
5. 保持高速且不摔会获得“老八稳住了”连击分。

关键 UI：

- 顶部：距离、分数、剩余时间。
- 底部或角色旁：平衡条。
- 摔车前提示：屏幕轻微晃动，角色角度夸张，音效/文字提示“稳住”。

美术策略：

- 使用 `assets/laoba-cutout-trimmed.png` 作为主角大图。
- 通过 CSS 或 Canvas 对整张角色图做旋转、轻微上下抖动、速度拖影。
- 车轮旋转可以先用额外的简化圆环图层模拟；后续如果想更精细，再拆轮子或做 spritesheet。
- 背景建议用校园砖路横向滚动，和原图场景呼应。

可玩性参数初稿：

- 倾斜角安全区：-12 到 12 度。
- 警告区：12 到 22 度。
- 摔车区：超过 26 度并持续 0.45 秒。
- 加速会提高分数倍率，但也提高随机晃动强度。
- 连续 5 秒不进入警告区，触发一次“稳如老八”奖励。
